# Prompts de IA utilizados - Producto 2

## 1. Introducción

Durante el desarrollo del Producto 2 de JobConnect se han utilizado herramientas de inteligencia artificial como apoyo para analizar requisitos, organizar la estructura del proyecto, mejorar la maquetación, refactorizar código y generar documentación técnica.

El uso de IA se ha planteado como una ayuda al aprendizaje y a la mejora del proyecto, pero manteniendo siempre la revisión manual de las decisiones tomadas y la adaptación al contexto real del enunciado.

## 2. Objetivos del uso de IA

Los principales objetivos del uso de IA en este producto han sido los siguientes:

- entender mejor el enunciado y la rúbrica
- reorganizar el código del Producto 1 hacia el Producto 2
- implementar persistencia en navegador
- mejorar la estructura modular del proyecto
- resolver errores de JavaScript
- mejorar la maquetación visual de la tabla de publicaciones
- preparar documentación UML y requerimientos funcionales
- redactar el README y otros documentos del proyecto

## 3. Prompts utilizados durante el desarrollo

### 3.1. Análisis del enunciado y planificación

Prompt utilizado:

Analiza el enunciado y la rúbrica del Producto 2 y dime qué funcionalidades hay que implementar exactamente, qué tecnologías pide y cómo reorganizar el proyecto a partir del Producto 1.

Utilidad:

Permitió descomponer el trabajo en pasos y detectar los módulos principales necesarios.

### 3.2. Reorganización modular del proyecto

Prompt utilizado:

A partir del Producto 1, dime cómo reorganizar los archivos JavaScript para el Producto 2 manteniendo una estructura limpia y modular.

Utilidad:

Sirvió para definir la nueva estructura basada en:

- `almacenaje.js`
- `int_1_dashboard.js`
- `int_2_login.js`
- `int_3_empleos.js`
- `int_4_usuarios.js`
- `ui.js`
- `datos-iniciales.js`

### 3.3. Persistencia con localStorage

Prompt utilizado:

Guíame paso a paso para implementar usuarios y sesión activa con localStorage en un proyecto JavaScript vanilla.

Utilidad:

Permitió desarrollar:

- alta de usuarios persistente
- almacenamiento del usuario activo
- mantenimiento de sesión
- cierre de sesión

### 3.4. Persistencia con IndexedDB

Prompt utilizado:

Explícame paso a paso cómo implementar IndexedDB para guardar y recuperar publicaciones en una aplicación web con JavaScript vanilla.

Utilidad:

Sirvió para implementar:

- apertura de la base de datos
- inicialización de publicaciones
- registro de nuevas publicaciones
- consulta de publicaciones
- borrado de publicaciones
- almacenamiento de selección del dashboard

### 3.5. Canvas

Prompt utilizado:

Ayúdame a crear un gráfico simple con Canvas que muestre el número de ofertas y demandas y que se actualice cuando cambien los datos.

Utilidad:

Permitió desarrollar el gráfico nativo de la página de ofertas y demandas.

### 3.6. Drag and Drop

Prompt utilizado:

Guíame paso a paso para implementar drag and drop en el dashboard de forma que se puedan seleccionar y deseleccionar publicaciones.

Utilidad:

Sirvió para construir:

- arrastre de tarjetas
- zona de selección
- persistencia de selección
- deselección arrastrando de vuelta

### 3.7. Mejora visual de la tabla

Prompt utilizado:

Mejora la maquetación de la tabla de publicaciones para que se vea profesional, ordenada, legible y sin problemas de distribución entre columnas.

Utilidad:

Ayudó a:

- reorganizar el contenido de columnas
- mejorar espaciados
- redimensionar botones
- mejorar el aspecto visual de la tabla

### 3.8. UML y documentación

Prompt utilizado:

Genera una propuesta clara de UML para un proyecto modular en JavaScript vanilla y ayúdame a redactar los requerimientos funcionales por componente.

Utilidad:

Sirvió para preparar:

- documento de requisitos
- diagrama de componentes
- diagrama UML de módulos y funciones

### 3.9. README del Producto 2

Prompt utilizado:

Redáctame un README completo y profesional adaptado al Producto 2, explicando funcionalidades, tecnologías, persistencia y estructura del proyecto.

Utilidad:

Permitió actualizar la documentación principal del repositorio.

## 4. Valoración del uso de IA

El uso de IA ha sido útil principalmente como apoyo para:

- estructurar mejor el trabajo
- reducir tiempo de organización
- recibir explicaciones paso a paso
- detectar errores
- mejorar la redacción técnica
- reforzar la documentación final

En todos los casos, el contenido generado ha requerido revisión y adaptación al proyecto real para asegurar que coincidiera con el enunciado y con la implementación final.

## 5. Conclusión

La inteligencia artificial ha sido utilizada como una herramienta de apoyo en el desarrollo del Producto 2 de JobConnect. Su uso ha contribuido a mejorar tanto la implementación técnica como la documentación y la organización general del proyecto, siempre complementado con revisión manual y adaptación al contexto del trabajo académico.
